Why does useEffect run twice in React 18 strict mode even with an empty dependency array?
Asked 2 hours ago 847 views 3 answers
P
Asked by priya_k
react hooks react-18 strict-mode

I upgraded my project to React 18 and noticed that my useEffect hook is being called twice during development, even when I pass an empty dependency array [].

Here's a minimal example of what I'm observing:

useEffect(() => {
  console.log('Effect ran');
  fetchUserData(); // This gets called twice!

  return () => {
    console.log('Cleanup ran');
  };
}, []);

The console shows: Effect ran → Cleanup ran → Effect ran. This is causing duplicate API calls and my analytics events are firing twice. Is this a bug? How do I fix it?

Answers 3
Accepted Answer
T
tanvir_r
· answered 1 hour ago
▲ 8

This is intentional behavior in React 18's Strict Mode — not a bug. React deliberately mounts, unmounts, and remounts components in development to help you catch side effects that are not properly cleaned up.

The fix is to ensure your effect is idempotent or use a ref to track whether the effect has already run:

useEffect(() => {
  let isMounted = true;

  async function fetchData() {
    if (!isMounted) return;
    await fetchUserData();
  }

  fetchData();

  return () => {
    isMounted = false; // Cleanup: cancel pending operations
  };
}, []);

For API calls specifically, use AbortController to properly cancel in-flight requests on cleanup.

Adding to the above answer — you can also disable StrictMode if you're in a legacy codebase and can't refactor immediately, but this is strongly discouraged for new code:

// Instead of:
<React.StrictMode>
  <App />
</React.StrictMode>

// Use temporarily:
<App />

But please treat this as a temporary workaround only. The double invocation is React telling you about real problems.
